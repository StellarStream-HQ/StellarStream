feat(compliance): Add comprehensive audit logging middleware and service

## Summary
Implement complete audit trail for all admin operations to meet HIGH priority compliance requirements. System captures all HTTP requests/responses, tracks data changes, and provides export/query functionality for compliance audits.

## Changes

### Database
- Add AdminAuditLog model with comprehensive fields
- Optimized indexes for performance (timestamp, userId, method, path)
- Migration script: add_admin_audit_log.sql

### Middleware
- New auditLogMiddleware: Captures request/response details
- Automatic client IP extraction (X-Forwarded-For support)
- Data sanitization: redacts passwords, API keys, credit cards
- Large payload truncation (>5KB)
- Helper function: setAuditContext() for custom audit data

### Services
- AdminAuditLogService with query, export, and archive methods
- Query filters: date range, user, method, path, status code, execution time
- Export to CSV and JSON formats
- Statistics calculation for compliance dashboard
- Archive and cleanup for retention policy

### API Routes
- GET /api/audit/logs - Query with comprehensive filters
- GET /api/audit/logs/user/:userId - User-specific queries
- GET /api/audit/logs/path - Path-specific queries
- GET /api/audit/logs/high-risk - Detect errors/slow requests
- GET /api/audit/statistics - Compliance dashboard
- POST /api/audit/export - Export to CSV/JSON
- POST /api/audit/archive - Archive and delete old records

### Tests
- 10 unit tests for auditLogMiddleware
- 13 unit tests for AdminAuditLogService
- 23 total test cases covering all scenarios

### Documentation
- AUDIT_LOG_COMPLIANCE.md - Comprehensive integration guide
- AUDIT_LOG_IMPLEMENTATION_SUMMARY.md - Implementation details
- AUDIT_LOG_QUICK_START.md - 5-minute setup guide

## Technical Details

### Security
- All sensitive data automatically redacted
- All endpoints require requireAdmin middleware
- Immutable append-only logs
- Graceful error handling

### Performance
- Optimized database indexes
- Non-blocking async writes
- Large payload truncation
- Batch export limit (10k records)

### Compliance
- Complete audit trail for all admin operations
- Field-level change tracking
- User identification and IP logging
- Export capabilities for auditors
- Retention policy support

## Integration Steps

1. Run migration: `npx prisma migrate dev`
2. Add middleware to Express app (after body parser)
3. Mount routes: `app.use(auditLogRoutes)`
4. Use setAuditContext() in admin routes
5. Set up retention cleanup job (optional)
